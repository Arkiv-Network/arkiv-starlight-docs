## Running the App

<Steps>

1. **Start the backend**

   In one terminal, run the data publisher:

   ```bash
   node --env-file backend/.env backend/index.js
   ```

   You should see entity keys logged for each token every 60 seconds.

2. **Serve the frontend**

   In another terminal:

   ```bash
   npx serve frontend/
   ```

3. **View the dashboard**

   Open `http://localhost:3000`. Within a minute you should see live price cards and chart updates.

</Steps>

<Aside>
Make sure you have test ETH from the [Kaolin Faucet](https://kaolin.hoodi.arkiv.network/faucet/) before running the backend.
</Aside>

## What You've Built

This tutorial demonstrated the core Arkiv full-stack pattern:

1. **Backend writes** — A Node.js service uses `createWalletClient` with a private key to batch-create entities via `mutateEntities()`
2. **Frontend reads** — A static HTML/JS page uses `createPublicClient` (no private key) to query entities by attribute and owner
3. **Time-scoped storage** — Entities expire after 3 hours, so old data is automatically cleaned up
4. **CDN imports** — The frontend loads the SDK directly from ESM without a build step

## Next Steps

- **Add more tokens** — Add additional cryptocurrency IDs to the backend's CoinGecko request and create corresponding chart containers in the frontend
- **Display more data** — Add trading volume, 7-day price changes, or other fields from the CoinGecko API
- **Filter by time range** — Use numeric `timestamp` attributes with `gt()` and `lt()` operators to show only recent data
- **Add a project attribute** — Follow the [best practices](/typescript-sdk/best-practices/) to add a project attribute for production use
- **Deploy** — Put the frontend on Vercel or Netlify and run the backend on a server or cloud function
